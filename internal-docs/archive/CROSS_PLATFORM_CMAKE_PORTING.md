### Cross‑Platform CMake Porting Guide – Phase 1 (macOS Executable)

This document describes **Phase 1** of making the app cross‑platform using **CMake only** (no NMake instructions here).

**Phase 1 goal**:  
- **Windows**: Migrate from Win32 windowing to **GLFW** (removing Win32-specific window code), while keeping DirectX 11 rendering and all existing functionality.
- **macOS**: Add a **stub GUI application** using **ImGui + GLFW + Metal** that can:
  - Load an index from file (using existing `IndexFromFilePopulator` infrastructure).
  - Display index statistics and details in a simple UI.
  - **No USN monitoring** on macOS (static index only).

Both platforms will share the same target name **`FindHelper`** (conditional sources based on platform).

**Note**: GLFW is already present in `external/glfw/` with headers (`include/GLFW/`) and Windows libraries (`lib/`).

### 2. Phase 1 Objective: Unified Target with Platform-Specific Sources

**Goal**: Use the same target name **`FindHelper`** for both Windows and macOS, with conditional sources and platform-specific backends.

- **On Windows**:
  - Migrate from Win32 windowing to **GLFW** (removing `WndProc`, `WNDCLASSEX`, Win32 message loop).
  - Keep DirectX 11 rendering (`imgui_impl_dx11.cpp`).
  - Keep all existing functionality (USN monitoring, search, etc.).
  - Keep all existing test targets and PGO logic intact.

- **On macOS**:
  - Build a **full-featured GUI** using **ImGui + GLFW + Metal** (`imgui_impl_glfw.cpp` + `imgui_impl_metal.mm`).
  - Focus on **index-from-file** loading (using `IndexFromFilePopulator`) via `--index-from-file` command-line argument.
  - **Phase 1**: Display index statistics and details in a simple UI (stub).
  - **Phase 2**: Achieve **feature parity** with Windows for all search and UI capabilities (full search UI, results table, settings, metrics, saved searches).
  - **No USN monitoring** (static index only, but full search functionality).

#### 3.3 Executable Definition (Unified Target Name)

```cmake
# Enable Objective-C++ on macOS (required for Metal backend)
if(APPLE)
    enable_language(OBJCXX)
endif()

# Define platform-specific sources
if(WIN32)
    set(APP_SOURCES ${APP_SOURCES_WIN32})
    set(IMGUI_RENDERER_BACKEND ${IMGUI_DIR}/backends/imgui_impl_dx11.cpp)
    set(APP_RESOURCES resource.rc app_icon.ico)
elseif(APPLE)
    set(APP_SOURCES ${APP_SOURCES_APPLE})
    set(IMGUI_RENDERER_BACKEND ${IMGUI_DIR}/backends/imgui_impl_metal.mm)
    set(APP_RESOURCES "")  # No .rc files on macOS
endif()

# Common ImGui sources
set(IMGUI_SOURCES_COMMON
    ${IMGUI_DIR}/imgui.cpp
    ${IMGUI_DIR}/imgui_draw.cpp
    ${IMGUI_DIR}/imgui_tables.cpp
    ${IMGUI_DIR}/imgui_widgets.cpp
    ${IMGUI_DIR}/backends/imgui_impl_glfw.cpp
)

# Unified executable (same target name for both platforms)
if(WIN32)
    add_executable(find_helper WIN32 ${APP_SOURCES} ${IMGUI_SOURCES_COMMON} ${IMGUI_RENDERER_BACKEND} ${APP_RESOURCES})
else()
    add_executable(find_helper ${APP_SOURCES} ${IMGUI_SOURCES_COMMON} ${IMGUI_RENDERER_BACKEND})
endif()

# Set output name to FindHelper (consistent across platforms)
set_target_properties(find_helper PROPERTIES OUTPUT_NAME "FindHelper")
```

### 4. Windows: Migrate `main_gui.cpp` from Win32 to GLFW

**File**: `main_gui.cpp`

**Changes required**:

1. **Add GLFW includes** (after Windows headers):
   ```cpp
   #define GLFW_EXPOSE_NATIVE_WIN32
   #include <GLFW/glfw3.h>
   #include <GLFW/glfw3native.h>
   ```

2. **Replace ImGui Win32 backend with GLFW backend**:
   - Replace `#include "imgui_impl_win32.h"` with `#include "imgui_impl_glfw.h"`
   - Remove `ImGui_ImplWin32_EnableDpiAwareness()` call (GLFW handles DPI awareness)

3. **Remove Win32 window creation code**:
   - Delete `WndProc` function and declaration
   - Delete `WNDCLASSEX` registration and `CreateWindow` calls
   - Delete `ShowWindow` and `UpdateWindow` calls

4. **Add GLFW window creation**:
   - Initialize GLFW: `glfwInit()` with error callback
   - Create window: `glfwCreateWindow(1280, 800, "Find Helper", nullptr, nullptr)` with `GLFW_CLIENT_API` set to `GLFW_NO_API`
   - Get native HWND: `HWND hwnd = glfwGetWin32Window(window)` (for DirectX initialization)
   - Set window size callback: `glfwSetWindowSizeCallback(window, WindowResizeCallback)` to handle DirectX swap chain resizing

5. **Replace Win32 message loop with GLFW event loop**:
   - Replace `PeekMessage` / `DispatchMessage` loop with:
     ```cpp
     while (!glfwWindowShouldClose(window)) {
         glfwPollEvents();
         // ... ImGui frame and rendering ...
     }
     ```

6. **Update ImGui backend calls**:
   - Replace `ImGui_ImplWin32_Init(hwnd)` with `ImGui_ImplGlfw_InitForOther(window, true)`
   - Replace `ImGui_ImplWin32_NewFrame()` with `ImGui_ImplGlfw_NewFrame()`
   - Replace `ImGui_ImplWin32_Shutdown()` with `ImGui_ImplGlfw_Shutdown()`

7. **Cleanup**:
   - Replace `DestroyWindow` / `UnregisterClass` with `glfwDestroyWindow(window)` and `glfwTerminate()`

**Note**: DirectX initialization and rendering remain unchanged; only the windowing layer switches to GLFW.

### 5. macOS: Create `main_mac.mm` Entry Point

**New file**: `main_mac.mm`

**Phase 1 responsibilities**:

1. **Initialize GLFW and Metal**:
   - GLFW initialization and window creation (`GLFW_CLIENT_API = GLFW_NO_API`)
   - Metal device and command queue creation
   - Attach `CAMetalLayer` to Cocoa window (via `glfwGetCocoaWindow`)

2. **Initialize ImGui with GLFW + Metal backends**:
   - `ImGui::CreateContext()`
   - `ImGui_ImplGlfw_InitForOther(window, true)`
   - `ImGui_ImplMetal_Init(device)`

3. **Load index from file** (Phase 1 focus):
   - Parse command-line arguments (reuse `CommandLineArgs.cpp`)
   - If `--index-from-file <path>` is provided:
     - Create `FileIndex` instance
     - Use `IndexFromFilePopulator` to load the index
     - Call `file_index.FinalizeInitialPopulation()`

4. **Display index statistics UI**:
   - Show application version (`APP_VERSION` from `GitVersion.h`)
   - Show index statistics:
     - Total indexed files count
     - Index memory usage (if available)
     - Index load status (loaded from file vs empty)
   - Simple UI to browse/index details (Phase 1 can be minimal: just stats)

5. **Render loop**:
   - Standard ImGui + Metal render loop:
     - `glfwPollEvents()`
     - Update Metal layer drawable size
     - `ImGui_ImplMetal_NewFrame(renderPassDescriptor)`
     - `ImGui_ImplGlfw_NewFrame()`
     - `ImGui::NewFrame()`
     - Render UI (index stats window)
     - `ImGui::Render()`
     - `ImGui_ImplMetal_RenderDrawData(...)`
     - Present and commit

6. **Cleanup**:
   - Shutdown ImGui backends
   - Destroy GLFW window and terminate GLFW

**Phase 1 UI scope**: Simple window showing:
- App version
- Index file path (if loaded)
- Index statistics (file count, etc.)
- Basic status messages

**Phase 2 UI scope** (feature parity with Windows):
- Full search UI (inputs, controls, filters)
- Search results table with sorting
- Settings window
- Metrics window
- Saved searches
- Keyboard shortcuts
- All search features (path patterns, filename patterns, extensions, time filters, etc.)

### 6. Command-Line Infrastructure for Index Loading

**Existing infrastructure** (already in `main_gui.cpp` on Windows):

- `CommandLineArgs.cpp` already supports `--index-from-file <path>`
- `IndexFromFilePopulator.cpp` provides `populate_index_from_file(path, file_index)`
- After loading, call `file_index.FinalizeInitialPopulation()`

**macOS integration**:
- Reuse `CommandLineArgs.cpp` (should be cross-platform)
- Reuse `IndexFromFilePopulator.cpp` (should be cross-platform)
- In `main_mac.mm`, parse command-line args and load index if `--index-from-file` is provided

**Example flow**:
```cpp
CommandLineArgs cmd_args = ParseCommandLineArgs(argc, argv);
if (!cmd_args.index_from_file.empty()) {
    FileIndex file_index;
    if (populate_index_from_file(cmd_args.index_from_file, file_index)) {
        file_index.FinalizeInitialPopulation();
        // Display index stats in UI
    }
}
```

- Create GLFW + Metal + ImGui setup

**Outcome**: Both platforms build `FindHelper` with shared GLFW windowing, platform-specific rendering (DX11 vs Metal), and macOS can load/display static indices from file.
