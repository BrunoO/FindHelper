# UX Review - 2026-01-06

## Executive Summary
- **UX Health Score**: 7/10
- **Top 3 Usability Issues**:
  1.  Search button is not always visible.
  2.  Clear button is not prominently displayed.
  3.  No visual feedback when a search is in progress.
- **Accessibility Gaps**:
  -   No keyboard shortcuts for common actions.
- **Quick Wins**:
  -   Make the Search button always visible.
  -   Make the Clear button more prominent.
  -   Add a loading indicator to show when a search is in progress.

## Findings

### High
- **Search Button Not Always Visible**
  - **Location**: `src/ui/SearchControls.cpp`
  - **Issue Type**: Discoverability & Learnability
  - **User Impact**: New users may not know how to initiate a search, as the `Search` button is only visible when the manual search section is expanded.
  - **Current Behavior**: The `Search` button is only visible when the `manualSearchExpanded` state is true.
  - **Recommended Solution**: Make the `Search` button always visible, regardless of the state of the manual search section.
  - **Severity**: High
  - **Effort**: Small

### Medium
- **Clear Button Not Prominently Displayed**
  - **Location**: `src/ui/SearchControls.cpp`
  - **Issue Type**: Discoverability & Learnability
  - **User Impact**: Users may not know how to clear the search filters, as the `Clear` button is not prominently displayed.
  - **Current Behavior**: The `Clear` button is a small button located next to the `Search` button.
  - **Recommended Solution**: Make the `Clear` button more prominent by increasing its size and adding a descriptive label.
  - **Severity**: Medium
  - **Effort**: Small

- **No Visual Feedback When Search Is in Progress**
  - **Location**: `src/ui/SearchControls.cpp`
  - **Issue Type**: Feedback & Status Communication
  - **User Impact**: Users may not know that a search is in progress, which can lead to confusion and frustration.
  - **Current Behavior**: There is no visual feedback to indicate that a search is in progress.
  - **Recommended Solution**: Add a loading indicator to show when a search is in progress.
  - **Severity**: Medium
  - **Effort**: Medium

### Low
- **No Keyboard Shortcuts for Common Actions**
  - **Location**: `src/core/ApplicationLogic.cpp`
  - **Issue Type**: Accessibility & Inclusivity
  - **User Impact**: Power users may be slowed down by the lack of keyboard shortcuts for common actions.
  - **Current Behavior**: There are no keyboard shortcuts for common actions like initiating a search or clearing the search filters.
  - **Recommended Solution**: Add keyboard shortcuts for common actions, such as `Ctrl+F` to focus the search input and `F5` to refresh the search.
  - **Severity**: Low
  - **Effort**: Medium
