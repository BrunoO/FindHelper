# UX Review - 2026-02-18

## Executive Summary
- **UX Health Score**: 9/10
- **Top 3 Usability Issues**:
  1. No visual indicator of current "Sort Column" besides the arrow (standard but could be better).
  2. "Clear All" button is adjacent to "Search now", risking accidental clearing.
  3. AI Search requires an environment variable, which isn't easy for non-technical users to set.
- **Total Findings**: 8
- **Estimated Remediation Effort**: 10 hours

## Findings

### Medium
1. **Accidental Clearing of Search Inputs**
   - **Location**: `src/ui/SearchInputs.cpp` (Extensions line)
   - **Issue Type**: User Workflow / Error Prevention
   - **User Impact**: Users might accidentally click "Clear All" instead of "Search now" as they are in the same vertical column of buttons.
   - **Recommended Solution**: Move "Clear All" to a less prominent location or add a subtle confirmation/undo for large clears.
   - **Severity**: Medium
   - **Effort**: Small (1 hr)

2. **AI Search Configuration Complexity**
   - **Location**: `src/ui/SearchInputs.cpp` (RenderAISearch)
   - **Issue Type**: Feature Discovery / Learnability
   - **User Impact**: Non-technical users won't know how to set an environment variable (`GEMINI_API_KEY`).
   - **Recommended Solution**: Add a field in the "Settings" menu to enter and persist the API key in the `settings.json` file.
   - **Severity**: Medium
   - **Effort**: Medium (4 hrs)

### Low
1. **Keyboard Shortcut Discoverability**
   - **Location**: Throughout the UI
   - **Issue Type**: Task Efficiency / Discoverability
   - **User Impact**: Many power-user shortcuts (dired marking, navigation) are only visible if you open the "Help" popup.
   - **Recommended Solution**: Add subtle shortcut hints in tooltips or labels (e.g., "Search now (Ctrl+Enter)").
   - **Severity**: Low
   - **Effort**: Small (2 hrs)

2. **Wait Cursor for Long Searches**
   - **Location**: Search Worker / UI interaction
   - **Issue Type**: Feedback & Status Communication
   - **User Impact**: During very large searches that take >500ms, the UI shows "Searching..." but the mouse cursor doesn't change, which can feel like the app is unresponsive.
   - **Recommended Solution**: Change the mouse cursor to a wait/busy cursor during active search operations.
   - **Severity**: Low
   - **Effort**: Small (1 hr)

## UX Health Score: 9/10
The application provides a professional, highly functional interface tailored to power users. The multiple UI modes (Full, Simplified, Minimalistic) are a standout feature that allows the app to scale from casual to professional use cases.

## Top 3 Usability Issues
1. **API Key Setup**: Too technical for casual users.
2. **Button Proximity**: "Search" vs "Clear All" placement.
3. **Shortcut Visibility**: Power features are hidden behind a Help button.

## Accessibility Gaps
- **Contrast**: ImGui default themes sometimes have low contrast; the project uses a custom `Theme.h` which improves this.
- **Screen Readers**: ImGui has limited support for standard screen readers.

## Quick Wins
1. **Add shortcut hints** to button tooltips.
2. **Add API Key field** to the Settings dialog.
3. **Highlight the "Search now" button** with a slightly more prominent color (already partially done with `Accent` color).

## Design System Recommendations
- Continue using the `Theme.h` centralized color system.
- Standardize on 8px/16px spacing increments (already mostly followed).
