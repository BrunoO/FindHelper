# UX Review - 2026-02-14

## Executive Summary
- **UX Health Score**: 7/10
- **Top Usability Issues**: 3
- **Accessibility Gaps**: 2
- **Quick Wins**: 4

## Findings

### High
- **Silent Overriding of User Preferences in Simplified/Minimalistic Modes**:
  - **Location**: `src/ui/SearchInputs.cpp`
  - **Issue Type**: User Workflow & Task Flow
  - **User Impact**: Users might be confused why "Instant Search" is active even if they disabled it in settings, as it is programmatically enforced in certain UI modes.
  - **Recommended Solution**: Show a message or disable the toggle in the settings window with a tooltip explaining "Enforced in Simplified Mode".
  - **Severity**: High

- **Search Syntax Discovery**:
  - **Location**: Search Input fields
  - **Issue Type**: Discoverability & Learnability
  - **User Impact**: Advanced prefixes like `rs:`, `pp:`, and `ex:` are powerful but not discoverable within the UI without reading external documentation.
  - **Recommended Solution**: Add a "Syntax Help" (?) icon next to the search box that opens a small popup or tooltip with examples.
  - **Severity**: High

### Medium
- **Truncated Path Readability**:
  - **Location**: `ResultsTable`
  - **Issue Type**: Screen Real Estate
  - **User Impact**: Middle-truncation or beginning-truncation can sometimes hide the most important part of a long path (the parent folders).
  - **Recommended Solution**: Provide a tooltip on hover with the full path.
  - **Severity**: Medium

### Low
- **Color Consistency for Metrics**:
  - **Location**: `MetricsWindow`
  - **Issue Type**: Consistency & Standards
  - **Recommended Solution**: Ensure consistent color coding for "Good" (Green), "Warning" (Yellow), and "Critical" (Red) performance metrics.
  - **Severity**: Low

## Quick Wins
- Add tooltips to all icon-only buttons.
- Implement `Enter` key support for all popup dialogs.
- Add a "Clear All Filters" button to the main search bar.

## Summary Requirements
- **UX Health Score**: 7/10 - Functional and efficient for power users, but lacks "on-ramps" for discoverability of advanced features.
- **Accessibility Gaps**: Keyboard navigation through the Results Table and Filter Panel could be improved.
