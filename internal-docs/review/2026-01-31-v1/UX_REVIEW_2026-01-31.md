## Findings

### Medium

#### 1. Modal Regex Generator Blocks Workflow
- **Location**: `src/ui/Popups.cpp` - `RenderRegexGeneratorPopup`
- **Issue Type**: User Workflow & Task Flow
- **User Impact**: Users cannot see the search results or adjust other filters while the regex generator is open. This prevents iterative testing of the regex against the visible file list.
- **Current Behavior**: Renders as a `BeginPopupModal`.
- **Recommended Solution**: Convert to a non-modal floating window or an expandable panel within the search area to allow concurrent interaction with the main UI.
- **Severity**: Medium
- **Effort**: Medium

#### 2. Hidden Keyboard Shortcuts
- **Location**: `src/ui/Popups.cpp` - `RenderKeyboardShortcutsPopup`
- **Issue Type**: Discoverability & Learnability
- **User Impact**: Power users may never discover the available keyboard shortcuts because they are buried in a modal popup that must be explicitly opened.
- **Current Behavior**: Shortcuts are only visible in a dedicated popup.
- **Recommended Solution**: Show common shortcuts (e.g., "Enter to search", "Del to delete") in tooltips or as subtle text hints in the empty state or footer.
- **Severity**: Medium
- **Effort**: Small

### Low

#### 3. Lack of Recent Search History
- **Location**: `src/ui/SearchInputs.cpp`
- **Issue Type**: Task Efficiency
- **User Impact**: Users must manually re-type or use the "Saved Searches" popup to repeat a recent query.
- **Current Behavior**: No history dropdown for the search input fields.
- **Recommended Solution**: Implement a "Recent Searches" dropdown that appears when the search field is focused or clicked.
- **Severity**: Low
- **Effort**: Medium

#### 4. "Memory: N/A" on Non-Windows Platforms
- **Location**: `src/ui/StatusBar.cpp:115`
- **Issue Type**: Consistency & Standards
- **User Impact**: Users on macOS/Linux see "Memory: N/A", which feels like a bug or missing feature.
- **Current Behavior**: Memory usage tracking is only implemented for Windows.
- **Recommended Solution**: Implement basic resident set size (RSS) tracking for POSIX platforms to provide consistent feedback.
- **Severity**: Low
- **Effort**: Small

#### 5. Ambiguous "Folders Only" Interaction
- **Location**: `src/ui/FilterPanel.cpp`
- **Issue Type**: Input Design & Interaction Patterns
- **User Impact**: It's not immediately clear if "Folders Only" affects the current search or if it's a global filter applied *after* search.
- **Current Behavior**: It's a toggle that triggers a new search.
- **Recommended Solution**: Group it more closely with the search inputs or provide a clearer visual connection to the search initiation.
- **Severity**: Low
- **Effort**: Small

## Summary Requirements

- **UX Health Score**: 8/10. The interface is clean, responsive, and well-suited for its target audience of power users. The status bar provides excellent system feedback.
- **Top 3 Usability Issues**:
  1. Modal nature of the Regex Generator.
  2. Discoverability of keyboard shortcuts.
  3. Lack of quick access to recent queries.
- **Accessibility Gaps**: The interface relies heavily on mouse interaction for certain features (like the regex generator templates). Ensure all popup actions are fully keyboard-navigable.
- **Quick Wins**:
  1. Add shortcut hints to search field tooltips.
  2. Implement RSS tracking for macOS/Linux to remove "Memory: N/A".
- **User Journey Assessment**: The "Search -> View -> Open" workflow is very efficient. The "I need a complex regex" workflow is slightly hindered by the modal generator.
