# UX Designer Review - 2026-02-20

## Executive Summary
- **UX Health Score**: 8/10
- **Top 3 Usability Issues**:
  1. **Advanced Search Discovery**: Regex and path patterns are powerful but hidden behind a small question-circle icon.
  2. **Inconsistent "Instant Search"**: The behavior of "Search as you type" changes significantly between UI modes (forced on in Simplified/Minimalistic, off by default in Full), which might confuse users who switch modes.
  3. **AI Search Friction**: The "Generate Prompt" / "Paste Response" flow for users without a Gemini API key is high-friction and could be better explained inline.

## Findings

### 1. Information Architecture & Layout

**Complexity in Full UI Mode**
- **Location**: Search Input Area
- **Issue Type**: Cognitive Load
- **User Impact**: New users might be overwhelmed by the number of input fields (Path, Extensions, Filename) and checkboxes.
- **Current Behavior**: All fields are shown by default in Full mode.
- **Recommended Solution**: Default to a single "Smart Search" box that parses common patterns (like extensions) automatically, with an "Advanced" toggle to show specific fields.
- **Severity**: Medium
- **Effort**: Large

### 2. Feedback & Status Communication

**Index Build Progress**
- **Location**: Status Bar / Empty State
- **Issue Type**: Status Information
- **User Impact**: During initial crawl, users see "Indexing..." but might not realize the app is fully usable for partial results.
- **Current Behavior**: Displays "Indexing [folder]... [N] entries".
- **Recommended Solution**: Use a more prominent progress bar and explicitly state "You can start searching now - results will update as indexing continues."
- **Severity**: Low
- **Effort**: Small

### 3. Accessibility & Inclusivity

**Keyboard Shortcut Discovery**
- **Location**: Entire Application
- **Issue Type**: Keyboard Navigation
- **User Impact**: Power users won't know about Ctrl+Enter or dired-style marking ('m', 'u', 't') without finding the Help window.
- **Current Behavior**: Shortcuts are documented in the Help window and some tooltips.
- **Recommended Solution**: Add a "Shortcuts" legend at the bottom of the results table or show hint overlays (e.g., "Press 'm' to mark") when a row is selected.
- **Severity**: Medium
- **Effort**: Medium

### 4. Consistency & Standards

**Primary Action Button**
- **Location**: `SearchInputs.cpp`
- **Issue Type**: Visual Hierarchy
- **User Impact**: The "Search now" button is colored with the "Accent" color, which is good. However, it's placed next to the "Path" input.
- **Current Behavior**: "Search now" is on the same line as the Path input.
- **Recommended Solution**: Ensure the primary action button is always in a consistent, prominent location, regardless of which input field has focus.
- **Severity**: Low
- **Effort**: Small

## Summary

- **UX Health Score**: 8/10. For a technical tool, the interface is clean, responsive, and makes good use of ImGui's strengths. The addition of "Simplified" and "Minimalistic" modes greatly improves accessibility for non-power users.
- **Top 3 Usability Issues**:
  1. High friction for AI search without API key.
  2. Discovery of powerful search features (Regex/Patterns).
  3. Visibility of keyboard shortcuts for dired-style operations.
- **Accessibility Gaps**:
  - Focus indicators in ImGui can sometimes be subtle.
  - No screen reader support (inherent to ImGui/GLFW).
- **Quick Wins**:
  - Add "Cmd/Ctrl + Enter to search" hint to the Search button tooltip.
  - Make the "Search Help" toggle more prominent.
  - Add a "Marking Help" tooltip to the 'Mark' column header.
- **Design System Recommendations**:
  - Standardize the use of the "Accent" color for all primary actions (Search, Save, Delete).
  - Use consistent padding/spacing between input groups.
- **User Journey Assessment**: The core journey (type -> results) is extremely fast and efficient. The secondary journeys (filter -> export) are also well-supported.
