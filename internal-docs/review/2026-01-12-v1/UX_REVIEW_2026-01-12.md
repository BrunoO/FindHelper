# UX Review - 2026-01-12

## Executive Summary
- **UX Health Score**: 6/10
- **Top 3 Usability Issues**:
    1.  **Confusing Information Hierarchy**: The AI-Assisted Search is given more prominence than the primary Manual Search, which is collapsed by default.
    2.  **Poor Feedback on Errors**: Network or API errors provide no feedback in the UI, leaving the user with a perpetual loading state.
    3.  **Inefficient Filter Management**: Clearing filters requires multiple clicks, and there is no way to quickly see all active filters at a glance.
- **Accessibility Gaps**: The application likely has poor keyboard navigation and lacks tooltips for icon-only buttons, making it difficult to use for people who rely on assistive technologies.

## Findings

### High
1.  **Location**: Main Window Layout
    -   **Issue Type**: Information Architecture & Layout / Visual Hierarchy
    -   **User Impact**: Users who want to perform a simple, manual file search (the most common use case) are presented with a collapsed "Manual Search" section and a prominent, expanded "AI-Assisted Search" section. This is confusing and inefficient for the primary user workflow.
    -   **Current Behavior**: The "Manual Search" section is collapsed by default.
    -   **Recommended Solution**: Reverse the default state. The "Manual Search" section should be expanded by default, and the "AI-Assisted Search" should be collapsed. The manual search is the core feature and should be immediately accessible.
    -   **Severity**: High
    -   **Effort**: S

2.  **Location**: AI-Assisted Search Panel
    -   **Issue Type**: Feedback & Status Communication / Error Messages
    -   **User Impact**: When an AI search fails due to a network error, invalid API key, or other backend issue, the user is not informed. The UI continues to show a "Generating..." state indefinitely.
    -   **Current Behavior**: The UI provides no feedback on network or API errors.
    -   **Recommended Solution**: The UI state must be updated to track and display error conditions. When an API call fails, the "Generating..." message should be replaced with a clear, user-friendly error message like "Could not connect to AI service. Please check your API key and network connection."
    -   **Severity**: High
    -   **Effort**: M

3.  **Location**: Filter Panel
    -   **Issue Type**: Input Design & Interaction Patterns / Filter & Control Design
    -   **User Impact**: To clear all active filters (e.g., time, size, extensions), the user must manually reset each one. There is no single "Clear All" or "Reset Filters" button.
    -   **Current Behavior**: Filters can only be cleared individually.
    -   **Recommended Solution**: Add a "Clear All Filters" button to the filter panel. This button should reset all filter controls to their default state.
    -   **Severity**: High
    -   **Effort**: S

### Medium
1.  **Location**: Search Inputs
    -   **Issue Type**: User Workflow & Task Flow / Task Efficiency
    -   **User Impact**: The three separate input fields for Path, Filename, and Extensions can be confusing. It is not immediately clear how they combine or that leaving the filename/extension fields empty triggers a full path search.
    -   **Recommended Solution**: Consolidate to a single, primary search bar for the main query. Use checkboxes or toggleable buttons for search modes (e.g., "Search File Names", "Search Paths", "Use Regex"). This simplifies the primary interface while keeping advanced options accessible.
    -   **Severity**: Medium
    -   **Effort**: M

2.  **Location**: Application-wide
    -   **Issue Type**: Accessibility & Inclusivity / Keyboard Navigation
    -   **User Impact**: Many controls, especially icon-only buttons, lack tooltips. This makes the interface difficult to learn and use for people who rely on screen readers or who are unfamiliar with the iconography.
    -   **Current Behavior**: Many buttons and controls have no hover text.
    -   **Recommended Solution**: Add `ImGui::IsItemHovered()` checks after creating controls and display a `ImGui::SetTooltip()` with a descriptive string for all icon-only buttons and complex input fields.
    -   **Severity**: Medium
    -   **Effort**: M

3.  **Location**: Results Table
    -   **Issue Type**: Performance Perception / Feedback During Operations
    -   **User Impact**: When a search yields a very large number of results, the application can become unresponsive for several seconds while it populates the results vector and prepares the table. The user is given no feedback during this time.
    -   **Current Behavior**: The UI freezes during the processing of large result sets.
    -   **Recommended Solution**: The search and result processing should be fully asynchronous. The UI should immediately show a loading indicator or a message like "Found 1,500,000 results. Preparing view...". The results should then be loaded into the view in batches to keep the UI responsive. (This is related to the performance issue of full result materialization).
    -   **Severity**: Medium
    -   **Effort**: L

### Quick Wins
1.  **Expand "Manual Search" by default**: A one-line change to `ImGui::CollapsingHeader` to improve the default user experience.
2.  **Add a "Clear All Filters" button**: A simple new button that calls the existing filter reset logic.
3.  **Add tooltips to the main action buttons**: Add tooltips for the "Search", "Clear", and other primary buttons to improve discoverability.
