## Findings

### High
1. **Confusing "Simplified UI" Mode Behavior**
   - **Location**: `src/ui/SearchInputs.cpp: Render`
   - **Issue Type**: User Workflow & Task Flow
   - **User Impact**: Users who prefer a clean interface but want manual control over when searches run (e.g., to avoid performance hits while typing) are forced into "Instant Search" mode when Simplified UI is enabled.
   - **Current Behavior**: `if (settings.simplifiedUI) { state.instantSearch = true; state.autoRefresh = true; }`. These settings are forced and hidden.
   - **Recommended Solution**: Decouple UI complexity from search behavior. Allow users to choose "Simplified UI" while still having access to (or preserving) their search behavior preferences.
   - **Severity**: High
   - **Effort**: S

### Medium
2. **Clunky AI Search Workflow without API Key**
   - **Location**: `src/ui/SearchInputs.cpp: RenderAISearch`
   - **Issue Type**: User Workflow & Task Flow
   - **User Impact**: Users without an API key must manually copy a generated prompt, paste it into an external AI, and then paste the result back. This is a high-friction workflow.
   - **Current Behavior**: Two-step manual copy-paste process.
   - **Recommended Solution**: Provide clearer "getting started" instructions for the API key or integrate with system-level AI tools (like Copilot on Windows) if possible to reduce steps.
   - **Severity**: Medium
   - **Effort**: M

### Low
3. **Overly Verbose Placeholders**
   - **Location**: `src/ui/SearchInputs.cpp: RenderPathSearchInput`
   - **Issue Type**: Form Design
   - **User Impact**: Placeholder text is so long it may be truncated in smaller window sizes, making it less helpful.
   - **Current Behavior**: `"e.g., C:\\projects\\src or regex pattern or path pattern"`
   - **Recommended Solution**: Shorten placeholders to "e.g., C:\path or regex" and use the Help button/tooltip for the full list of supported patterns.
   - **Severity**: Low
   - **Effort**: S

4. **Inconsistent Keyboard Shortcut Documentation**
   - **Location**: `src/ui/SearchInputs.cpp: RenderPathSearchInput (Tooltip)`
   - **Issue Type**: Discoverability & Learnability
   - **User Impact**: Users might not realize that Ctrl+Enter works globally, not just in the path field.
   - **Current Behavior**: Tooltip for "Search now" button mentions the shortcut, but it's not prominently displayed elsewhere.
   - **Recommended Solution**: Add a small "Shortcuts" legend or ensure all input tooltips mention relevant global shortcuts consistently.
   - **Severity**: Low
   - **Effort**: S

## Summary Requirements

### UX Health Score: 7/10
Justification: The interface is functional and follows standard power-user patterns. However, the coupling of UI modes with search logic and the friction in the AI workflow are significant usability hurdles.

### Top 3 Usability Issues
1. Forced "Instant Search" in Simplified UI.
2. High-friction AI search without API key.
3. Lack of visible shortcut key cues in the main interface.

### Accessibility Gaps
- Tab order has not been explicitly audited for the custom ImGui layout.
- Screen reader support is limited by the immediate-mode GUI framework.

### Quick Wins
1. Remove forced `instantSearch` in Simplified UI mode.
2. Shorten input placeholders.
3. Add " (Ctrl+Enter)" text to the Search button label.

### User Journey Assessment
The "I know what I'm looking for" journey is well-supported. The "Help me find this" journey (AI search) is significantly degraded without an API key, which might be the case for many first-time users.
