# UX Review - 2026-02-19

## Executive Summary
- **UX Health Score**: 8/10
- **Top 3 Usability Issues**:
  1. Discovery of keyboard shortcuts.
  2. Visual density in Manual Search section.
  3. Indication of "Total Files" calculation progress.
- **Total Findings**: 7
- **Estimated Remediation Effort**: 16 hours

## Findings

### High
1. **Shortcut Discoverability**
   - **Location**: Results Table
   - **Issue Type**: Discoverability & Learnability: Feature Discovery
   - **User Impact**: Users may not know that 'm', 'u', 't', and 'Shift+D' work for bulk operations until they read the documentation.
   - **Current Behavior**: Shortcuts work but are only documented in a separate help window.
   - **Recommended Solution**: Add a small legend or toolbar button tooltip that mentions these shortcuts when a row is selected.
   - **Effort**: Small

2. **Manual Search Section Density**
   - **Location**: Main Window
   - **Issue Type**: Information Architecture & Layout: Visual Hierarchy
   - **User Impact**: The "Manual Search" section contains many controls (inputs, filters, controls) which can be overwhelming.
   - **Current Behavior**: All manual search controls are visible when expanded.
   - **Recommended Solution**: Use sub-tabs or better grouping within the Manual Search section to reduce immediate cognitive load.
   - **Effort**: Medium

### Medium
1. **Partial Results Indication**
   - **Location**: Status Bar
   - **Issue Type**: Feedback & Status Communication: Status Information
   - **User Impact**: During a very broad search, users might be confused why the result count keeps increasing.
   - **Current Behavior**: Shown in status bar, but could be more prominent.
   - **Recommended Solution**: Add a subtle "streaming" animation (e.g., a pulsing dot) near the result count.
   - **Effort**: Small

2. **Dired-style Navigation Feedback**
   - **Location**: Results Table
   - **Issue Type**: User Workflow & Task Flow: Task Efficiency
   - **User Impact**: When using 'n' or 'p' to navigate, the scroll position might jump unexpectedly if not handled carefully.
   - **Current Behavior**: `clipper.IncludeItemByIndex` and `ImGui::SetScrollHereY` are used, which is good.
   - **Recommended Solution**: Ensure the "selection highlight" is clearly distinct from "hover highlight" to avoid confusion during rapid keyboard navigation.
   - **Effort**: Small

### Low
1. **Truncated Path Tooltip Delay**
   - **Location**: Full Path Column
   - **Issue Type**: Feedback & Status Communication: Visual Feedback
   - **User Impact**: Users have to wait for the system tooltip delay to see the full path.
   - **Current Behavior**: Standard ImGui tooltip.
   - **Recommended Solution**: Provide a way to expand the column or show a faster tooltip for truncated items.
   - **Effort**: Small

## Summary Requirements

- **UX Health Score**: 8/10. The interface is highly functional and caters well to its target audience of power users. The addition of "UI Modes" (Full, Simplified, Minimalistic) is a great UX win.
- **Top 3 Usability Issues**:
  1. Shortcut discovery for bulk actions.
  2. Visual clutter in the manual search panel.
  3. Real-time feedback for long-running metadata loads.
- **Accessibility Gaps**: Text contrast is generally good (standard ImGui Dark/Light), but screen reader support is limited by the immediate-mode paradigm.
- **Quick Wins**:
  1. Add shortcut tooltips to the "Marked Actions" toolbar.
  2. Implement a pulsing animation for active searches.
- **Design System Recommendations**: Standardize the "Accent Tint" usage across all headers and primary buttons for a more cohesive brand feel.
- **User Journey Assessment**: The "AI-Assisted Search" provides an excellent entry point for new users, while the keyboard-driven "Dired-style" operations provide high efficiency for returning power users.
