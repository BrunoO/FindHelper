# UX Review - 2026-02-09

## Executive Summary
- **UX Health Score**: 8/10
- **Top 3 Usability Issues**:
  1. Implicit settings overrides in Simplified UI
  2. Hidden AI search status messages
  3. No visual distinction for "rs:" prefix patterns
- **Total Findings**: 10
- **Estimated Remediation Effort**: 12 hours

## Findings

### High
- **Location**: `src/ui/SearchInputs.cpp` - Simplified UI Mode
- **Issue Type**: Cognitive Load / Error Tolerance
- **User Impact**: Users who enable "Simplified UI" have their "Instant Search" and "Auto Refresh" preferences silently overridden. This can be confusing if the user expects one behavior but gets another without explanation.
- **Current Behavior**: Settings are forced to `true` in the code without notifying the user or updating the checkbox state in the settings window.
- **Recommended Solution**: Add a tooltip or information icon in the settings window explaining that Simplified UI enables these features by default.
- **Severity**: High
- **Effort**: S

### Medium
- **Location**: `src/ui/SearchInputs.cpp` - AI Search Status
- **Issue Type**: Feedback & Status Communication
- **User Impact**: AI search status messages (like "Generating...") and error messages are sometimes displayed at the bottom of the input area, which can be missed by users focused on the result list.
- **Current Behavior**: Text messages appear below buttons after a delay.
- **Recommended Solution**: Use a clear loading spinner or progress bar overlaid on the input field or results table when an AI search is in progress.
- **Severity**: Medium
- **Effort**: M

### Low
- **Location**: Search Input Field
- **Issue Type**: Discoverability & Learnability
- **User Impact**: The "rs:" prefix for regex and "pp:" for path patterns are powerful but not visually distinct from the query string.
- **Current Behavior**: Text is rendered in the standard color.
- **Recommended Solution**: Implement syntax highlighting in the search input field to color-code prefixes (`rs:`, `pp:`) differently from the pattern itself.
- **Severity**: Low
- **Effort**: L (requires custom ImGui input logic)

## Summary Requirements

- **UX Health Score**: 8/10. The application provides a very responsive and clean interface for a system utility. The use of FontAwesome icons and clear groupings makes it intuitive for power users.
- **Top 3 Usability Issues**:
  1. **Silent Overrides**: Simplified UI behavior is too opaque.
  2. **Feedback Visibility**: AI generation status is too subtle.
  3. **Discoverability**: Advanced search syntax depends on remembering prefixes.
- **Accessibility Gaps**:
  - Tabbing through the interface is not fully optimized; focus order can be unpredictable in the filter panel.
  - No screen reader specific labels for custom icon-only buttons.
- **Quick Wins**:
  1. Add tooltips explaining prefix syntax (`rs:`, `pp:`) when the user starts typing.
  2. Update "Simplified UI" description in settings.
- **Design System Recommendations**: Standardize the "reserved space" on the right of input fields for action buttons to prevent layout shifting when buttons appear/disappear.
- **User Journey Assessment**: The core journey (Search -> View Results -> Open File) is extremely efficient. The AI journey (Prompt -> Generate -> Apply) is well-integrated but could benefit from better progress visualization.
