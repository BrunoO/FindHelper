# UX Review - 2026-01-21

## Executive Summary
- **UX Health Score**: 4/10
- **Top 3 Usability Issues**: Unintuitive default layout, poor feedback for long-running operations, and inconsistent interaction patterns.
- **Accessibility Gaps**: The application is not keyboard-navigable and lacks basic accessibility features like text labels for icons.

The application's user experience is functional for a power-user but suffers from a lack of polish and several unintuitive design choices. The default layout hides the primary search functionality, and the system provides poor feedback during critical background operations like indexing. The immediate-mode nature of ImGui has led to some inconsistent and non-standard UI behaviors.

## Findings

### High
1.  **Unintuitive Default Layout**
    - **Location:** Main application window
    - **Issue Type:** Information Architecture & Layout / Visual Hierarchy
    - **User Impact:** New users are confused because the primary "Manual Search" section is collapsed by default, while the "AI-Assisted Search," which is a secondary feature, is expanded. This forces users to perform an extra click to access the most common functionality.
    - **Current Behavior:** The "AI-Assisted Search" section is expanded by default.
    - **Recommended Solution:** Change the default state so that the "Manual Search" section is expanded and the "AI-Assisted Search" section is collapsed. The most frequently used feature should be the most prominent.
    - **Severity:** High
    - **Effort:** S

2.  **Poor Feedback During Indexing**
    - **Location:** Main application window / Status Bar
    - **Issue Type:** Feedback & Status Communication / Visual Feedback
    - **User Impact:** The application provides minimal feedback while the initial file index is being built. The user sees a static "Indexing..." message but has no sense of progress or how long the operation will take. This can make the application feel frozen or unresponsive, especially during a long initial scan.
    - **Current Behavior:** A simple, non-updating text message is displayed during indexing.
    - **Recommended Solution:** Implement a progress bar in the status bar that shows the percentage of the file system that has been scanned. Additionally, display the number of files indexed so far (e.g., "Indexed 1,234,567 files").
    - **Severity:** High
    - **Effort:** M

3.  **Inconsistent Keyboard Interaction**
    - **Location:** Search input fields
    - **Issue Type:** Input Design & Interaction Patterns / Keyboard Shortcuts
    - **User Impact:** Users expect to be able to initiate a search by pressing the Enter key after typing in a search field. However, this is not implemented consistently. In some cases, it works; in others, the user must explicitly click the "Search" button.
    - **Current Behavior:** Pressing Enter in a search field does not reliably trigger a search.
    - **Recommended Solution:** Ensure that pressing the Enter key in any of the search input fields triggers the search action, just as clicking the "Search" button does. This is a standard convention that will improve task efficiency.
    - **Severity:** High
    - **Effort:** S

### Medium
1.  **Lack of Keyboard Navigation**
    - **Location:** Entire application
    - **Issue Type:** Accessibility & Inclusivity / Keyboard Navigation
    - **User Impact:** Users who rely on keyboards for navigation (due to disability or preference) cannot use the application effectively. It is impossible to move focus between UI elements using the Tab key.
    - **Current Behavior:** There is no support for keyboard navigation.
    - **Recommended Solution:** ImGui supports keyboard navigation. Enable and configure it so that users can use the Tab and arrow keys to navigate between all interactive elements in a logical order.
    - **Severity:** Medium
    - **Effort:** L

2.  **Missing Confirmation for Destructive Actions**
    - **Location:** Settings Window
    - **Issue Type:** Error Prevention & Recovery / Prevention
    - **User Impact:** Users can accidentally trigger a full rebuild of the index by clicking a single button in the settings, with no confirmation dialog. This can lead to the loss of the current index and a long wait for the rebuild, with no way to undo the action.
    - **Current Behavior:** Clicking the "Rebuild Index" button immediately starts the process.
    - **Recommended Solution:** Before any destructive action, such as rebuilding the index or clearing settings, display a modal confirmation dialog that clearly explains the consequences and requires the user to confirm their intent.
    - **Severity:** Medium
    - **Effort:** S

## Quick Wins
1.  **Change the default expanded section** from "AI-Assisted Search" to "Manual Search". This is a one-line code change that dramatically improves the initial user experience.
2.  **Implement the Enter key shortcut for search.** This is a small code change that aligns the application with standard user expectations.
3.  **Add a confirmation dialog for the "Rebuild Index" button.** This is a low-effort change that prevents a frustrating user error.
