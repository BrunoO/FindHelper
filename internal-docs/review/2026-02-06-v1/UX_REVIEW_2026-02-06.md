# UX Review - 2026-02-06

## Executive Summary
- **Health Score**: 7/10
- **Total Findings**: 8
- **Estimated Remediation Effort**: 12 hours

## Findings

### High
1. **Simplified UI Lacks Transparency**
   - **Location**: `SearchInputs.cpp` / Settings Window
   - **Issue Type**: Consistency & Standards / User Control
   - **User Impact**: Users enabling "Simplified UI" have their "Instant Search" and "Auto Refresh" settings changed and hidden without their knowledge. This leads to unexpected application behavior that the user cannot easily revert.
   - **Current Behavior**: Simplified UI silently overrides and hides other search settings.
   - **Recommended Solution**: Instead of hiding/overriding, "Simplified UI" should be a preset that *recommends* or *defaults* these settings but doesn't lock them. If locked, a message should explain why.
   - **Severity**: High
   - **Effort**: Medium

2. **Discoverability of Advanced Search Syntax**
   - **Location**: Search Input Fields
   - **Issue Type**: Discoverability & Learnability
   - **User Impact**: Power-user features like regex (`rs:`) or specific filters are hidden behind prefixes that are hard to discover without documentation.
   - **Current Behavior**: Users must know the `rs:` or `f:` prefixes to use advanced features.
   - **Recommended Solution**: Add a "Search Help" icon (?) next to the search box that opens a quick reference popup showing available prefixes and examples.
   - **Severity**: High
   - **Effort**: Small

### Medium
1. **Visual Density of Results Table**
   - **Location**: Results Table
   - **Issue Type**: Information Architecture & Layout
   - **User Impact**: In "Show All" or large result sets, the table can become a "wall of text" that is hard to scan.
   - **Current Behavior**: Standard ImGui table with many columns.
   - **Recommended Solution**: Use alternating row colors (striping) and ensure enough padding between columns. Highlight the matching part of the filename in search results.
   - **Severity**: Medium
   - **Effort**: Small

2. **Inconsistent Keyboard Navigation**
   - **Location**: Main Window
   - **Issue Type**: Accessibility & Inclusivity
   - **User Impact**: Navigating between Filename, Path, and Extension inputs using Tab is possible but the focus order or "Return to Search" behavior can be inconsistent.
   - **Current Behavior**: Basic ImGui tab focus.
   - **Recommended Solution**: Implement a custom focus loop that ensures the Filename input is always the primary focus on Ctrl+F or application start.
   - **Severity**: Medium
   - **Effort**: Small

### Low
1. **Missing Tooltips for Quick Filters**
   - **Location**: Filter Panel
   - **Issue Type**: Input Design & Interaction Patterns
   - **User Impact**: Icons for filters (e.g., "Folders Only") might not be immediately obvious.
   - **Current Behavior**: Icon buttons without hover text.
   - **Recommended Solution**: Add `ImGui::SetItemTooltip` to all filter buttons.
   - **Severity**: Low
   - **Effort**: Small

## Quick Wins
1. **Add tooltips to all icon buttons**: Immediate improvement in discoverability.
2. **Add "Search Help" popup**: Make advanced syntax accessible without documentation.
3. **Enable table striping**: Improve readability of search results.

## UX Health Score: 7/10
The application provides a powerful tool for developers and power users. Its performance-first approach reflects well in the UI (responsive even during heavy searches). However, it relies heavily on "hidden" knowledge (prefixes, magic settings) that increases the learning curve for new users.

## User Journey Assessment
- **New User**: Can easily perform a simple filename search, but might be confused by the multiple input boxes.
- **Power User**: Will appreciate the speed and regex support but might find the "Simplified UI" behavior frustrating if they want a clean UI with specific advanced settings.

## Design System Recommendations
- **Standardize Padding**: Ensure consistent spacing between the search controls and the results table.
- **Color Palette**: Use a more distinct "Accent" color for active filters and search highlights to improve scannability.
